Overview

A high-altitude balloon (HAB) carries a small payload to the edge of space — typically 80,000 to 100,000+ feet (24–30 km). At that altitude, you're above 99% of Earth's atmosphere. The sky is black. The curvature of the Earth is visible. And your sensors are recording data from an environment that most people will never experience.

The concept is simple: a large latex weather balloon filled with helium lifts a lightweight payload. As it rises, the decreasing atmospheric pressure causes the balloon to expand. Eventually — usually around 90,000–100,000 feet — the balloon bursts. The payload descends under a parachute, and you recover it using GPS tracking.

This is exactly what Michael's team did at NASA SEES. And you can do it from your backyard (well, a large open field).

What You'll Measure

Temperature (drops to -60°C in the stratosphere), barometric pressure (drops to near-vacuum), altitude via GPS (watching your payload climb in real time), and optionally: UV intensity, humidity, acceleration, and photographs of the curvature of the Earth.

FAA Regulations

Read This Before Anything Else

In the United States, unmanned free balloons are regulated by FAR 101 (Federal Aviation Regulations, Part 101). The good news: small HAB launches are exempt from most regulations if you meet certain criteria. But you must understand the rules.

Your launch is exempt from most FAR 101 requirements if ALL of the following are true:

  • Total payload weight (all packages combined) is under 6 pounds (2.72 kg).
  • No individual package weighs more than 4 pounds (1.81 kg).
  • The payload package has a weight-to-size ratio of less than 3 oz per square inch on any surface.
  • You use a radar reflector (easy to make from foil) so ATC radar can see you.
  • The rope/line between balloon and payload has a break strength of 50 lbs or less.
  • You do NOT launch within 5 miles of an airport boundary without ATC coordination.
Best Practice

Even though small payloads are technically exempt, it's courteous and safer to file a NOTAM (Notice to Air Missions) before launch. Call your local Flight Service Station at 1-800-WX-BRIEF. Tell them your launch site coordinates, estimated time, expected max altitude, and estimated landing zone. It takes 5 minutes and pilots will appreciate it.

Building the Payload

Wire the Flight Computer

Connect the BMP280, GPS module, and SD card module to the Arduino Nano:

WIRING: Arduino Nano ├── 3V3 ──→ BMP280 VCC ├── GND ──→ BMP280 GND ──→ GPS GND ──→ SD GND ├── A4 ──→ BMP280 SDA (I2C data) ├── A5 ──→ BMP280 SCL (I2C clock) ├── D4 ──→ GPS TX (software serial RX) ├── D3 ──→ GPS RX (software serial TX) ├── D10 ──→ SD CS (chip select) ├── D11 ──→ SD MOSI ├── D12 ──→ SD MISO ├── D13 ──→ SD SCK ├── 5V ──→ GPS VCC, SD VCC └── VIN ──→ 9V battery + (through toggle switch)

Solder Permanent Connections

This payload will experience vibration, extreme cold, and a hard landing. Breadboard connections WILL shake loose. Solder all connections to a protoboard (perfboard). Use hot glue on the solder joints for strain relief. This is not optional — a loose wire at 80,000 feet means lost data.

Test on the Ground

Before flight, verify every sensor works. Power up the payload. Check that the SD card is logging data (pull the card and read the file). Take it for a drive — verify the GPS is getting a fix and logging latitude/longitude. Put it in the freezer for an hour — verify the BMP280 still reads at -18°C (that's as cold as a home freezer gets; the real thing will be 3x colder, but this tests basic cold tolerance).

Flight Computer Code

// Launch Lab — HAB Flight Computer v1.0 // Logs: time, lat, lon, altitude, temperature, pressure to SD card #include <Wire.h> #include <SPI.h> #include <SD.h> #include <Adafruit_BMP280.h> #include <SoftwareSerial.h> #include <TinyGPS++.h> const int SD_CS = 10; const int GPS_RX = 4, GPS_TX = 3; Adafruit_BMP280 bmp; SoftwareSerial gpsSerial(GPS_RX, GPS_TX); TinyGPSPlus gps; File logFile; unsigned long logInterval = 2000; // Log every 2 seconds unsigned long lastLog = 0; void setup() { Serial.begin(9600); gpsSerial.begin(9600); if (!bmp.begin(0x76)) { Serial.println("BMP280 not found!"); while (1); } if (!SD.begin(SD_CS)) { Serial.println("SD card failed!"); while (1); } logFile = SD.open("flight.csv", FILE_WRITE); if (logFile) { logFile.println("time_ms,lat,lon,gps_alt_m,bmp_temp_c,bmp_pres_hpa"); logFile.flush(); } pinMode(LED_BUILTIN, OUTPUT); Serial.println("Flight computer ready."); } void loop() { while (gpsSerial.available() > 0) { gps.encode(gpsSerial.read()); } if (millis() - lastLog >= logInterval) { lastLog = millis(); logData(); } } void logData() { float temp = bmp.readTemperature(); float pres = bmp.readPressure() / 100.0; // hPa double lat = gps.location.isValid() ? gps.location.lat() : 0.0; double lon = gps.location.isValid() ? gps.location.lng() : 0.0; double alt = gps.altitude.isValid() ? gps.altitude.meters() : 0.0; String line = String(millis()) + "," + String(lat, 6) + "," + String(lon, 6) + "," + String(alt, 1) + "," + String(temp, 1) + "," + String(pres, 1); logFile = SD.open("flight.csv", FILE_WRITE); if (logFile) { logFile.println(line); logFile.flush(); logFile.close(); } // Blink LED to confirm logging digitalWrite(LED_BUILTIN, HIGH); delay(50); digitalWrite(LED_BUILTIN, LOW); Serial.println(line); }
Library Installation

In Arduino IDE, go to Sketch → Include Library → Manage Libraries. Install: "Adafruit BMP280" (by Adafruit), "TinyGPSPlus" (by Mikal Hart), "SD" (built-in). These are the only three external libraries you need.

Gondola Construction

Prepare the Styrofoam Cooler

The cooler is your gondola — it insulates the electronics from -55°C stratospheric cold and cushions the landing impact. Trim it to the smallest size that fits your payload. Cut a small hole in the lid for the GPS antenna (it needs a clear view of the sky). Seal all seams with tape.

Mount Electronics Inside

Use foam cutouts to create a snug cradle for the Arduino board and battery. Nothing should rattle. Place two hand warmers next to the battery (cold kills battery voltage). Place two more near the electronics. The warmers last 8+ hours — more than enough for a 2–3 hour flight.

Build the Radar Reflector

Cut three squares of cardboard (30cm × 30cm). Wrap each in aluminum foil (shiny side out). Cut slots in each square halfway through the center — one from the top, one from the bottom, one from the side — and interlock them into a 3D cross shape. Attach to the payload cord above the gondola. This reflects ATC radar and makes your payload visible to air traffic control.

Build or Buy the Parachute

An 18-inch parachute gives a descent rate of roughly 5 m/s for a 500g payload — fast enough to limit drift, slow enough for a soft landing. You can buy a premade ripstop nylon chute, or make one from a trash bag cut into an octagon with 8 string lines. Tie the shroud lines to a central ring, and attach that to the payload cord.

FLIGHT TRAIN (from top to bottom): 🎈 BALLOON (1200g latex, filled with helium) │ │ ← 10m nylon cord │ ═══ RADAR REFLECTOR (foil cross) │ │ ← 3m cord │ ☂️ PARACHUTE (18" — folded during ascent, │ opens after balloon bursts) │ │ ← 5m cord │ 📦 GONDOLA (styrofoam cooler with payload) Total flight train length: ~18m Total payload weight: aim for under 500g (well under FAR 101 limits)

Launch Day Procedure

Pre-Launch (Day Before)

  • Check weather forecast. You need: clear skies, winds under 15 mph at surface, no thunderstorms within 100 miles.
  • Run a prediction at predict.habhub.org — enter your launch site, balloon type, payload weight, and ascent rate. It shows the predicted flight path and landing zone.
  • Confirm your landing zone isn't a lake, military base, or dense urban area. Shift your launch time if needed.
  • Charge all batteries. Format the SD card. Run a final ground test of all sensors.

Launch Site Setup

Find a large open field — football fields, farm fields, or parks work. You need at least 100m clearance in all directions from trees, power lines, and buildings. Lay out the entire flight train on the ground before filling the balloon.

Fill the Balloon

Attach the balloon's neck to the helium tank nozzle. Fill slowly — the balloon should be roughly 1.5m (5 feet) in diameter at launch (it'll expand to 6+ meters before bursting). You need enough helium to provide positive "free lift" — the balloon must pull upward with enough force to lift the payload PLUS extra (typically aim for 1–2 kg of free lift). Use a kitchen scale: tie the filled balloon to a weight on the scale; the scale should read 1–2 kg less than its weight with balloon detached.

Final Checks

  • Power on the payload. Verify LED blinks (confirms logging).
  • Verify GPS lock (check serial output if you have a laptop handy).
  • Activate hand warmers and seal the gondola.
  • Tie the flight train: balloon → cord → reflector → cord → parachute → cord → gondola.
  • Double-check every knot. Add a cable tie and tape to each one.

Launch

Have someone hold the balloon while you walk the flight train out straight downwind. On "3, 2, 1" — release the balloon. It should climb quickly. You'll lose visual sight within 2–5 minutes. Your payload is now on its own.

Get in your car and start driving toward the predicted landing zone. The balloon will take 1.5–2.5 hours to reach burst altitude and another 30–45 minutes to descend.

Tracking & Recovery

Your GPS module logs position data to the SD card — but that only helps AFTER you recover the payload. For real-time tracking during flight, you have options:

  • Budget option: Use a cheap GPS tracker with cellular connectivity (like a Tile or similar — but these lose signal above cell tower range at ~5,000 ft). They'll re-acquire signal during descent below 5,000 ft, giving you the landing location.
  • Better option: SPOT Trace satellite tracker (~$50 used, plus monthly subscription). Works everywhere, even at altitude. Sends GPS position every 5 minutes via satellite.
  • DIY option: Add an APRS (amateur radio) transmitter. Requires a ham radio license (free to get, take a test). Transmits GPS coordinates continuously. This is what serious HAB teams use.
Recovery Tips

Write your phone number in large letters on the gondola with "HARMLESS SCIENCE PROJECT — PLEASE CALL" in case someone finds it before you do. Bring binoculars for the final approach. The parachute is usually visible from 200+ meters in an open field. Most payloads land within 5 miles of the predicted zone if your wind data was good.

Data Analysis

Your CSV File

After recovery, pull the SD card. Open flight.csv. You should have thousands of rows of data: timestamp, latitude, longitude, GPS altitude, temperature, pressure. This is real atmospheric science data.

Key Graphs to Plot

  • Altitude vs. Time: Shows your ascent rate, burst altitude, and descent profile. Should look like an inverted V.
  • Temperature vs. Altitude: Shows the troposphere/stratosphere temperature profile. Temperature drops through the troposphere, stabilizes at the tropopause (~36,000 ft), then may rise slightly in the stratosphere due to ozone absorption of UV.
  • Pressure vs. Altitude: Should follow the barometric formula — an exponential decay. Compare your data to the standard atmosphere model.
  • GPS Track on Map: Plot lat/lon on Google Maps or Google Earth. See the wind patterns that pushed your payload.

Going Further

Add a Camera

A cheap action camera (or old smartphone in airplane mode with camera set to timelapse) records stunning footage. You'll capture the transition from blue sky to black, the curvature of the Earth, and the balloon burst. This footage alone is enough to go viral or anchor a science presentation.

Add UV and Humidity Sensors

A UV index sensor (VEML6075, ~$4) measures how UV intensity changes with altitude and ozone concentration. A DHT22 humidity sensor (~$3) tracks moisture through the troposphere. More sensors = richer dataset = stronger research project.

Fly a Science Experiment

Use the balloon as a platform for an actual experiment. Examples: test how cosmic radiation affects film (put a piece of photographic film in the gondola — it'll show particle tracks). Test seed germination after stratospheric exposure. Fly yeast cultures and check if they survive. This is how real atmospheric science missions work — the balloon is just a delivery vehicle.

Enter a Competition

HAB projects are incredibly competitive entries for science fairs, NASA SEES, and the Conrad Challenge. The visual drama (photos from 100,000 feet), the engineering rigor (sensor calibration, data analysis), and the scale of the project impress every judge. Document everything from day one.

Overview

Every real propulsion lab — from NASA's Stennis Space Center to SpaceX's test facility in McGregor — has one thing in common: a thrust stand. It measures how much force an engine produces over time. The thrust curve tells you everything: total impulse, specific impulse, burn time, consistency, and efficiency.

You're going to build the same thing for $25. Your thrust logger uses a strain gauge load cell (the same sensor in your kitchen scale) connected to an Arduino via an HX711 amplifier. It reads force at 10–80 samples per second and logs every reading to your computer over serial. From there, you can graph thrust curves, calculate total impulse, and compare different motors or experiments.

Pairs With Everything

This logger works with model rocket motors, ionic wind thrusters (P-001), compressed air nozzles, rubber-band launchers — anything that produces a force. Build it once, use it forever.

How Load Cells Work

A load cell is a metal bar with tiny strain gauges bonded to its surface. When force is applied, the bar bends microscopically. The strain gauges (thin resistive foils) stretch with it, changing their resistance. Four gauges are wired in a Wheatstone bridge configuration — an elegant circuit that converts tiny resistance changes into a measurable voltage difference.

The problem: that voltage difference is incredibly small — microvolts. Your Arduino's built-in ADC (10-bit, 0–5V range) can't read it. That's where the HX711 comes in. It's a 24-bit analog-to-digital converter specifically designed for load cells. It amplifies the signal by 128x and digitizes it with 24-bit resolution — that's over 16 million discrete levels compared to your Arduino's 1,024.

WHEATSTONE BRIDGE (inside the load cell): +V (Excitation) │ ┌───┤───┐ │ R1 R2 ← Strain gauges │ │ │ ├───A B───┤ ← Measure voltage A-B │ │ │ (changes with force) │ R3 R4 │ │ │ └───┴────┘ │ GND When the bar bends: R1 and R4 increase (tension side) R2 and R3 decrease (compression side) → Voltage between A and B changes proportionally to force

Wiring the Circuit

Connect Load Cell to HX711

Your load cell has 4 wires (colors may vary — check your datasheet):

  • Red → HX711 E+ (Excitation+)
  • Black → HX711 E- (Excitation-)
  • White → HX711 A- (Signal-)
  • Green → HX711 A+ (Signal+)

Connect HX711 to Arduino

Only 4 wires needed:

  • VCC → Arduino 5V
  • GND → Arduino GND
  • DT (Data) → Arduino Pin 3
  • SCK (Clock) → Arduino Pin 2
COMPLETE WIRING DIAGRAM: LOAD CELL HX711 BOARD ARDUINO UNO ┌──────────┐ ┌──────────────┐ ┌───────────┐ │ Red ────┼──────────────→│ E+ │ │ │ │ Black ───┼──────────────→│ E- │ │ │ │ White ───┼──────────────→│ A- VCC │←───────────┤ 5V │ │ Green ───┼──────────────→│ A+ GND │←───────────┤ GND │ └──────────┘ │ DT │←───────────┤ Pin 3 │ │ SCK │←───────────┤ Pin 2 │ └──────────────┘ └───────────┘ Total connections: 8 wires. That's it.
Wire Color Warning

Load cell wire colors are NOT standardized across all manufacturers. If your readings are erratic or always zero, you may have the signal wires swapped. Try swapping white and green. If that doesn't work, try swapping red and black. A wrong excitation polarity won't damage anything — it just inverts the reading.

Arduino Code

Install the HX711 library: in Arduino IDE, go to Sketch → Include Library → Manage Libraries → search "HX711" → install the one by "bogde".

// Launch Lab — Thrust Logger v1.0 // Reads load cell via HX711 at ~10 Hz, outputs CSV over serial #include "HX711.h" const int DT_PIN = 3; const int SCK_PIN = 2; HX711 scale; // Calibration factor — you'll determine this in the calibration step. // Start with 1.0, then adjust based on known weight test. float calibration_factor = -7050.0; void setup() { Serial.begin(9600); Serial.println("time_ms,force_g"); // CSV header scale.begin(DT_PIN, SCK_PIN); scale.set_scale(calibration_factor); scale.tare(); // Zero the scale // Blink LED to signal ready pinMode(LED_BUILTIN, OUTPUT); for (int i = 0; i < 3; i++) { digitalWrite(LED_BUILTIN, HIGH); delay(200); digitalWrite(LED_BUILTIN, LOW); delay(200); } } void loop() { if (scale.is_ready()) { float force = scale.get_units(1); // Single reading, no averaging unsigned long t = millis(); Serial.print(t); Serial.print(","); Serial.println(force, 2); // 2 decimal places } delay(100); // ~10 Hz sample rate. Use delay(50) for ~20 Hz }

Calibration

Calibration converts raw HX711 readings into real-world grams. Without it, your numbers are meaningless.

Mount the Load Cell

Screw one end of the bar load cell to a rigid base (wood block, acrylic plate). The other end should be free to deflect. The arrow on the load cell (if present) indicates the direction of applied force.

Tare (Zero) the Scale

Upload the code above. Open Serial Monitor at 9600 baud. With nothing on the load cell, you should see readings near 0.00. If not, the tare in setup() should handle it. Press the Arduino reset button to re-tare.

Apply Known Weight

Place a known weight on the free end of the load cell. US coins are surprisingly precise calibration weights:

  • Nickel = 5.000g
  • Quarter = 5.670g
  • 10 nickels stacked = 50.00g

Read the value on Serial Monitor. If 50g reads as "35.72", your calibration factor needs adjusting.

Adjust Calibration Factor

Use this formula:

new_factor = old_factor × (displayed_value / actual_weight) Example: Old factor: -7050.0 Displayed: 35.72g Actual: 50.00g new_factor = -7050.0 × (35.72 / 50.00) = -5034.6 Update calibration_factor in the code, re-upload, and test again. Repeat until your known weight reads within ±0.1g.

Running a Test

Set Up Data Capture

You need to save the serial output to a file. Three options:

  • Arduino Serial Monitor: Copy-paste the output after the test (works for short tests).
  • CoolTerm: Free serial terminal. Set "Connection → Capture to Text File" before starting.
  • Python script: Use pyserial to read and save automatically (see Going Further).

Prepare the Experiment

Mount whatever you're testing so its thrust pushes directly on the load cell's free end. The force direction must align with the load cell's measurement axis. For model rocket motors, build a simple bracket that holds the motor with the nozzle pointed away from the load cell.

Record Data

Start your serial capture. Wait 2–3 seconds for baseline (should read ~0). Ignite/activate your experiment. Let it run to completion. Wait another 2–3 seconds of baseline after thrust ends. Stop capture.

You now have a CSV file with columns: time_ms, force_g

Data Analysis

Quick Spreadsheet Analysis

Open your CSV in Google Sheets or Excel. Create an XY scatter chart: X axis = time (ms), Y axis = force (g). You just plotted a thrust curve. Key measurements:

  • Peak thrust: The maximum force reading.
  • Average thrust: Mean of all readings during the burn.
  • Burn time: Duration where force > baseline noise.
  • Total impulse: Area under the thrust curve (integrate using trapezoidal rule in your spreadsheet).

Python Graphing (Optional)

import csv import matplotlib.pyplot as plt time_ms = [] force_g = [] with open('thrust_data.csv', 'r') as f: reader = csv.DictReader(f) for row in reader: time_ms.append(float(row['time_ms'])) force_g.append(float(row['force_g'])) # Convert ms to seconds time_s = [t / 1000.0 for t in time_ms] plt.figure(figsize=(10, 5)) plt.plot(time_s, force_g, color='#00b4ff', linewidth=1.5) plt.fill_between(time_s, force_g, alpha=0.15, color='#00b4ff') plt.xlabel('Time (s)') plt.ylabel('Force (g)') plt.title('Thrust Curve') plt.grid(True, alpha=0.3) plt.tight_layout() plt.savefig('thrust_curve.png', dpi=150) plt.show()

Troubleshooting

ProblemLikely CauseFix
Readings stuck at 0DT/SCK pins swapped, or load cell wires wrongDouble-check wiring. Try swapping DT and SCK. Try swapping white/green load cell wires.
Readings fluctuate wildly (±5g)Loose connections; electrical noiseSecure all connections. Use shorter wires. Keep load cell wires away from motors/power lines. Add 0.1µF capacitor across VCC-GND on HX711.
Readings drift over timeTemperature change; load cell creepLet the system warm up 5 min before testing. Re-tare (press reset) right before each test run.
Serial Monitor shows garbageWrong baud rateSet Serial Monitor to 9600 baud (must match Serial.begin in code).
Values are negativeForce applied in wrong direction; wires swappedFlip the load cell, or swap the signal wires (green/white), or negate the calibration factor (change sign).
Very slow reading (once per 2+ seconds)HX711 in 10Hz mode (default is 10 SPS)Some HX711 boards have a RATE pin. Connect it to VCC for 80 SPS. Check your board documentation.

Going Further

Increase Sample Rate to 80 Hz

The HX711 has a RATE pin. By default (pin floating or grounded) it samples at 10 Hz. Connect the RATE pin to VCC (5V) and it jumps to 80 Hz. Change your delay(100) to delay(12) in the code. This gives you much smoother thrust curves and catches fast transients.

SD Card Logging

Add a micro-SD card module (~$3) to log data directly to a file — no computer tether needed. Useful for field tests like model rocket launches. Use the Arduino SD library.

Real-Time Serial Plotter

Arduino IDE has a built-in Serial Plotter (Tools → Serial Plotter). Just output one number per line (remove the CSV header and timestamp) and you get a live graph. Great for real-time visualization during tuning.

Multi-Axis Measurement

Add a second load cell perpendicular to the first to measure lateral forces. This is how real test stands check for thrust vectoring asymmetry. You'll need a second HX711 board.

Compare Commercial Rocket Motors

Buy a few different Estes model rocket motors (A, B, C class). Measure their thrust curves with your logger. Compare your data to the manufacturer's published curves. If they match — your instrument is validated. If they don't — figure out why. That's engineering.
